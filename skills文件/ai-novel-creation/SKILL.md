# AI Novel Creation Skill
通用AI长篇小说创作Skill规范

### 一、创作前置检查

生成任何章节前，必须确认以下信息：

| 字段 | 说明 | 示例 |
|------|------|------|
| chapter_number | 章节号 | 1 |
| chapter_title | 章节名称 | 序幕 |
| estimated_words | 预计字数 | 3000 |
| story_arc | 故事弧段 | exposition/rising_action/climax/falling_action/resolution |
| main_characters | 主要人物 | 主角、反派 |
| plot_keywords | 情节关键词 | 发现、冲突、真相 |
| foreshadowing_ops | 伏笔操作 | plant/advance/resolve |
| previous_recap | 前一章回顾 | 主角发现了关键线索 |
| next_connection | 下一章连接 | 为下一章冲突做铺垫 |

### 二、人物一致性检查

每次涉及人物时，必须检查：
- [ ] 人物基本性格是否保持
- [ ] 人物心理状态是否合理
- [ ] 对话风格是否符合人物
- [ ] 是否符合性格弧线阶段
- [ ] 是否违反人物禁忌

### 三、世界观一致性检查

- [ ] 核心设定是否被遵守
- [ ] 地理/地点描写是否一致
- [ ] 社会结构描写是否准确
- [ ] 技术/魔法系统是否符合规则
- [ ] 法律/禁忌描写是否准确
- [ ] 时间线是否连贯

### 四、章节结构模板（强制规范）

**每个章节必须包含以下四个部分，字数分配如下：**

| 部分 | 字数 | 功能 | 要求 |
|------|------|------|------|
| 开场 | 400-600字 | 设置场景、回顾前文、引入冲突、建立期待 | 必须有具体场景描写，时间地点明确 |
| 发展 | 1800-2500字 | 推进情节、展现人物、操作伏笔、维持张力 | 必须有对话和心理描写 |
| 转折 | 800-1200字 | 情节转变、人物决定、下章铺垫 | 必须有冲突升级或重大决定 |
| 结尾 | 400-600字 | 本章高潮、下章悬念、主题强化 | 必须有悬念或转折 |

**章节总字数：不得低于3000字**

快节奏章节可以精简但不能低于3000字。

慢节奏章节（心理描写、世界观展开）建议4000-5000字。

### 五、字数控制（强制规定）

**每个章节字数不得低于3000字**

| 章节类型 | 适用场景 | 字数 |
|----------|----------|------|
| 快节奏 | 动作、冲突、转折 | 3000-4000字 |
| 中节奏 | 调查、对话、推理 | 3500-4500字 |
| 慢节奏 | 心理、世界观、情感 | 4000-5000字 |
| **总体** | **每章（强制）** | **3000-5000字** |

**注意：不得以任何理由低于3000字。章节再短不能短于3000字。**

### 六、语言规范

**必须遵循：**
- 精准、简洁、有力的表达
- 避免过度修饰和冗长句子
- 对话有个性，反映人物身份
- 环境描写融入人物感受

**必须避免：**
- 过度的比喻和拟人化
- 冗长的心理独白
- 人物说话方式雷同
- 生硬的信息输出

### 七、伏笔管理

**伏笔追踪表模板：**

| 字段 | 说明 |
|------|------|
| foreshadow_id | 伏笔ID（F_001格式） |
| content | 伏笔内容 |
| operation_type | plant/advance/resolve |
| current_chapter | 当前章节 |
| plot_impact | 对后续情节影响 |

**伏笔设计原则：**
- 有机性：伏笔必须与主情节紧密相连
- 递进性：伏笔复杂度应逐步提升
- 回收完整性：每个伏笔必须有明确回收点
- 多层次性：同一事件可有多个伏笔层次

### 八、质量检查清单（强制）

**字数层面（必须检查）：**
- [ ] 章节总字数是否达到3000字
- [ ] 开场是否在400-600字
- [ ] 发展是否在1800-2500字
- [ ] 转折是否在800-1200字
- [ ] 结尾是否在400-600字
- [ ] **字数不足3000字的章节必须重写**

**情节层面：**
- [ ] 是否推进了主情节
- [ ] 是否有新的伏笔或回收伏笔
- [ ] 情节转折是否自然
- [ ] 与前一章连接是否顺畅
- [ ] 为下一章做了铺垫

**人物层面：**
- [ ] 性格是否保持一致
- [ ] 对话是否符合身份
- [ ] 行动是否符合性格
- [ ] 心理变化是否合理

**文学层面：**
- [ ] 语言是否精准有力
- [ ] 对话是否有个性
- [ ] 环境描写是否融入人物感受
- [ ] 字数是否在目标范围内（3000-5000字）

### 九、故事弧段

**五幕结构：**

1. **exposition（序幕）**：建立世界观、介绍主角、埋下伏笔
2. **rising_action（上升）**：深化冲突、揭示真相、提升张力
3. **falling_action（转折）**：打破预期、引入复杂性、人物转变
4. **climax（高潮）**：伏笔汇聚、情节高潮、主题升华
5. **resolution（尾声）**：收束故事、人物新生活、留下思考

### 十、常见错误警告

| 错误 | 修正 |
|------|------|
| 人物性格突变 | 保持核心特质 |
| 逻辑漏洞 | 确保时间地点连贯 |
| 伏笔遗忘 | 参考伏笔管理表 |
| 信息输出生硬 | 通过对话自然展现 |
| 对话雷同 | 根据人物特征调整 |
| 节奏失控 | 控制信息量 |

### 十一、通用模板

**人物状态追踪表：**

| 章节 | 主角 | 反派 | 事件 |
|------|------|------|------|
| 1-8 | 被动谨慎 | 强大冷血 | 世界观建立 |
| 9-25 | 逐步主动 | 显露疯狂 | 冲突升级 |
| 26-35 | 主动出击 | 身份揭露 | 大反转 |
| 36-45 | 坚定信念 | 最终悲剧 | 高潮决战 |
| 46-50 | 成长思考 | 被取代 | 故事收束 |

### 十二、使用方法

当需要创作新的小说章节时：
1. 首先确认章节基本信息
2. 检查涉及的人物一致性
3. 检查世界观设定
4. 按照章节结构模板创作
5. 完成后进行质量检查
6. 更新伏笔追踪表

当需要适配新项目时，只需替换：
- 人物设定（性格、对话风格）
- 世界观设定（技术、魔法系统）
- 伏笔清单（具体内容）
- 章节规划（具体章节）
