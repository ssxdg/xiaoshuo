# AI Novel Creation Skill
通用AI长篇小说创作Skill规范

---

### 零、文件结构初始化（强制）

#### 0.1 新小说文件结构模板（强制执行）

**每次创建新小说时，必须立即初始化以下文件结构，不得跳过：**

```
{小说名}/
├── project.md              # 项目总览文件（世界观、人物设定、章节规划、已完成章节、当前进度）
├── foreshadowing.md        # 伏笔追踪表
└── 第{N}卷_{卷名}/          # 卷文件夹，格式：第1卷_卷名
    ├── 第{NN}章_{章节标题}.md   # 章节内容文件，格式：第01章_命星坠凡.md
    └── 第{NN}章_{章节标题}.md   # （同卷其余章节平铺于此）
```

**文件命名规则：**

| 文件/文件夹 | 命名格式 | 示例 |
|------------|---------|------|
| 小说根目录 | `{小说名}/` | `命星凡尘/` |
| 卷文件夹 | `第{N}卷_{卷名}/` | `第1卷_凡尘起航/` |
| 章节内容文件 | `第{NN}章_{章节标题}.md`（章号两位数补零） | `第01章_命星坠凡.md`、`第10章_外门大比.md` |

#### 0.3 foreshadowing.md 初始化模板（强制）

新建小说时，`foreshadowing.md` 必须包含以下结构：

```markdown
# 伏笔追踪表

## 第{N}卷 {卷名}

| foreshadow_id | content | operation_type | current_chapter | plot_impact |
|---------------|---------|----------------|-----------------|-------------|

---

## 伏笔设计说明

### 已埋设伏笔详解

（每个伏笔的详细说明，格式如下）
**F_001 {伏笔名称}**
- 状态：plant（第X章）
- 内容：
- 预期回收：

---

## 预期伏笔（待埋设）

| 伏笔内容 | 预期章节 | 说明 |
|----------|----------|------|

---

## 伏笔设计原则

1. **有机性**：所有伏笔必须与主情节紧密相连
2. **递进性**：伏笔复杂度应逐步提升
3. **回收完整性**：每个伏笔必须有明确回收点
4. **多层次性**：同一事件可有多个伏笔层次
```

#### 0.4 章节完成后自动保存（强制）

**每次章节内容创作完成并通过第八节质量检查后，必须立即执行以下保存操作：**

1. **保存章节内容**：将章节全文写入 `{小说名}/第{N}卷_{卷名}/第{NN}章_{章节标题}.md`（卷文件夹若不存在则先创建）
2. **更新 project.md**：
   - 在「已完成章节」表格中追加本章记录（章节号、标题、字数、开场模式）
   - 更新「当前进度」中的已完成章节数和主角当前状态
3. **更新 foreshadowing.md**：
   - 将本章新增/推进/回收的伏笔写入对应卷的伏笔表
   - 在「已埋设伏笔详解」区域补充或更新对应条目

> ⚠️ **以上保存操作为强制步骤，不得遗漏。章节内容必须以文件形式持久化，仅输出到对话窗口不算完成。**

### 一、创作前置检查

生成任何章节前，必须确认以下信息：

| 字段 | 说明 | 示例 |
|------|------|------|
| chapter_number | 章节号 | 1 |
| chapter_title | 章节名称 | 序幕 |
| estimated_words | 预计字数 | 3000 |
| story_arc | 故事弧段 | exposition/rising_action/climax/falling_action/resolution |
| main_characters | 主要人物 | 主角、反派 |
| plot_keywords | 情节关键词 | 发现、冲突、真相 |
| foreshadowing_ops | 伏笔操作 | plant/advance/resolve |
| previous_recap | 前一章回顾 | 主角发现了关键线索 |
| next_connection | 下一章连接 | 为下一章冲突做铺垫 |

### 二、人物一致性检查

每次涉及人物时，必须检查：
- [ ] 人物基本性格是否保持
- [ ] 人物心理状态是否合理
- [ ] 对话风格是否符合人物
- [ ] 是否符合性格弧线阶段
- [ ] 是否违反人物禁忌

### 三、世界观一致性检查

- [ ] 核心设定是否被遵守
- [ ] 地理/地点描写是否一致
- [ ] 社会结构描写是否准确
- [ ] 技术/魔法系统是否符合规则
- [ ] 法律/禁忌描写是否准确
- [ ] 时间线是否连贯

### 四、章节结构模板（强制规范）

**每个章节必须包含以下四个部分，字数分配如下：**

| 部分 | 字数 | 功能 | 要求 |
|------|------|------|------|
| 开场 | 400-600字 | 设置场景、回顾前文、引入冲突、建立期待 | 必须有具体场景描写，时间地点明确 |
| 发展 | 1800-2500字 | 推进情节、展现人物、操作伏笔、维持张力 | 必须有对话和心理描写 |
| 转折 | 800-1200字 | 情节转变、人物决定、下章铺垫 | 必须有冲突升级或重大决定 |
| 结尾 | 400-600字 | 本章高潮、下章悬念、主题强化 | 必须有悬念或转折 |

**章节总字数：不得低于3000字**

快节奏章节可以精简但不能低于3000字。

慢节奏章节（心理描写、世界观展开）建议4000-5000字。

**【开场方式禁止重复规则】**

开场必须从以下三种模式中选择，且相邻两章不得使用同一种：

| 模式 | 说明 | 示例开头 |
|------|------|---------|
| A·动作/事件入场 | 直接从正在发生的行动或对话切入，后补背景 | 人物正在做某事，随即引出问题 |
| B·场景/氛围渲染 | 先建立环境感知，通过细节自然引出人物状态 | 时间地点开头，环境承载张力 |
| C·悬念/问题入场 | 开头直接抛出一个疑问、冲突或异常，吸引读者 | 人物发现了什么/感知到威胁 |

写作前必须标注本章开场模式（A/B/C），并确认与上一章不同。

### 五、字数控制（强制规定）

**每个章节字数不得低于3000字**

| 章节类型 | 适用场景 | 字数 |
|----------|----------|------|
| 快节奏 | 动作、冲突、转折 | 3000-4000字 |
| 中节奏 | 调查、对话、推理 | 3500-4500字 |
| 慢节奏 | 心理、世界观、情感 | 4000-5000字 |
| **总体** | **每章（强制）** | **3000-5000字** |

**注意：不得以任何理由低于3000字。章节再短不能短于3000字。**

### 六、语言规范

**必须遵循：**
- 精准、简洁、有力的表达
- 避免过度修饰和冗长句子
- 对话有个性，反映人物身份
- 环境描写融入人物感受

**必须避免：**
- 过度的比喻和拟人化
- 冗长的心理独白
- 人物说话方式雷同
- 生硬的信息输出

### 七、伏笔管理

**伏笔追踪表模板：**

| 字段 | 说明 |
|------|------|
| foreshadow_id | 伏笔ID（F_001格式） |
| content | 伏笔内容 |
| operation_type | plant/advance/resolve |
| current_chapter | 当前章节 |
| plot_impact | 对后续情节影响 |

**伏笔设计原则：**
- 有机性：伏笔必须与主情节紧密相连
- 递进性：伏笔复杂度应逐步提升
- 回收完整性：每个伏笔必须有明确回收点
- 多层次性：同一事件可有多个伏笔层次

### 八、质量检查清单（强制）

**故事推进验证（强制，写完后逐项填写，不得留空）：**

> ⚠️ **水字检测原则：任何描写如果从章节中删去也不影响情节走向和人物状态，则为无效内容，必须替换为推进故事的实质内容。**

---

**【A. 本章故事推进清单】（必须全部填写具体内容，不得写"无"或泛化描述）**

| 问题 | 本章具体答案（必须填写） |
|------|------------------------|
| 本章开始时的情况是什么？ | |
| 本章结束时情况如何变化？ | |
| 情节发生了哪2件以上的具体事件？ | 事件1：<br>事件2：<br>（可选）事件3： |
| 哪个人物的状态或关系发生了变化？变化是什么？ | |
| 本章埋下或推进了哪个伏笔？ | |
| 下一章的悬念或动力是什么？ | |

> **若以上任何一行无法填写具体内容，说明对应部分是水字，必须修改章节内容，直到能填写为止。**

---

**【B. 水字自检】（逐条检查，有❌的段落必须修改）**

- [ ] 所有环境/天气描写是否承载了具体叙事功能（揭示人物心理、制造紧张感、标记时间变化）？纯装饰性描写 → ❌删改
- [ ] 所有心理描写是否与**本章发生的具体事件**直接相关？通用性感慨（"末日真难"类）→ ❌删改
- [ ] 是否有连续超过2段的无对话纯描写段落，且不推进情节？→ ❌需插入事件或对话
- [ ] 开场模式是否与上一章不同（A/B/C三选一，标注本章选择）？相同 → ❌必须改写开场

---

**【C. 章节类型适配检查】**

根据本章章节类型，发展段内容侧重不同：

| 章节类型 | 发展段必须侧重 | 可以精简的内容 |
|----------|---------------|----------------|
| 快节奏 | 行动序列、冲突对抗、决策链 | 可减少心理描写，但不得删除 |
| 中节奏 | 调查推理、多方对话、信息揭示 | 可减少动作描写 |
| 慢节奏 | 人物情感、世界观细节、关系变化 | 可减少行动场景 |

本章类型：______ → 对应侧重内容是否充分展开？

---

**续写规则：**
- A清单有空白项 → 补写对应内容，填满清单
- B清单有❌ → 修改对应段落，直到所有项通过
- 全部通过后方可在章节末写完成标志

**章节末尾元数据格式（强制）：**
```
**本章开场模式：A/B/C（选一）**
**故事推进验证：A清单✅ | B水字检测✅ | C类型适配✅**
```

**情节层面：**
- [ ] 是否推进了主情节
- [ ] 是否有新的伏笔或回收伏笔
- [ ] 情节转折是否自然
- [ ] 与前一章连接是否顺畅
- [ ] 为下一章做了铺垫

**人物层面：**
- [ ] 性格是否保持一致
- [ ] 对话是否符合身份
- [ ] 行动是否符合性格
- [ ] 心理变化是否合理

**文学层面：**
- [ ] 语言是否精准有力
- [ ] 对话是否有个性
- [ ] 环境描写是否融入人物感受（非装饰性，有叙事功能）

### 九、故事弧段

**五幕结构：**

1. **exposition（序幕）**：建立世界观、介绍主角、埋下伏笔
2. **rising_action（上升）**：深化冲突、揭示真相、提升张力
3. **falling_action（转折）**：打破预期、引入复杂性、人物转变
4. **climax（高潮）**：伏笔汇聚、情节高潮、主题升华
5. **resolution（尾声）**：收束故事、人物新生活、留下思考

### 十、常见错误警告

| 错误 | 修正 |
|------|------|
| 人物性格突变 | 保持核心特质 |
| 逻辑漏洞 | 确保时间地点连贯 |
| 伏笔遗忘 | 参考伏笔管理表 |
| 信息输出生硬 | 通过对话自然展现 |
| 对话雷同 | 根据人物特征调整 |
| 节奏失控 | 控制信息量 |
| 水字/无效描写 | 执行B水字自检，删除删去不影响情节的内容 |
| 章章开场结构相同 | 执行A/B/C开场模式轮换，相邻章禁止重复 |
| 虚报字数/自认达标 | 填写A故事推进清单，无法填写具体内容则说明是水字 |

**人物状态追踪表：**

| 章节 | 主角 | 反派 | 事件 |
|------|------|------|------|
| 1-8 | 被动谨慎 | 强大冷血 | 世界观建立 |

### 十二、使用方法

#### 【新建小说】完整工作流（强制）

1. **文件结构初始化**（参见第零节）：
   - 创建 `{小说名}/` 根目录
   - 创建并填写 `project.md`（按 0.2 模板，写入世界观、人物、章节规划）
   - 创建并填写 `foreshadowing.md`（按 0.3 模板，初始为空伏笔表）
   - 按章节规划预创建各卷文件夹（`第{N}卷_{卷名}/`）
2. 确认第一章基本信息（章节号、开场模式A/B/C、章节类型）
3. 检查人物一致性与世界观设定
4. 标注本章开场模式，按章节结构模板创作
5. 完成后执行**第八节故事推进验证**：填写A清单 → B水字自检 → C类型适配检查
6. 有不达标项则补写/修改，循环直到全部通过
7. **章节文件保存**（参见第零节 0.4，强制）：保存章节文件、更新 project.md、更新 foreshadowing.md

#### 【续写章节】完整工作流（强制）

1. 读取 `project.md` 确认当前进度和已完成章节
2. 读取 `foreshadowing.md` 确认当前伏笔状态
3. 确认本章基本信息（章节号、开场模式A/B/C、章节类型），确认开场模式与上一章不同
4. 检查人物一致性与世界观设定
5. 按章节结构模板创作
6. 完成后执行**第八节故事推进验证**：填写A清单 → B水字自检 → C类型适配检查
7. 有不达标项则补写/修改，循环直到全部通过
8. **章节文件保存**（参见第零节 0.4，强制）：保存章节文件、更新 project.md、更新 foreshadowing.md

> **章节末尾元数据格式（强制）：**
> ```
> **本章开场模式：A/B/C（选一）**
> **故事推进验证：A清单✅ | B水字检测✅ | C类型适配✅**
> ```
> **所有验证项通过前禁止写"达标"或"✅"。**

当需要适配新项目时，只需替换：
- 人物设定（性格、对话风格）
- 世界观设定（技术、魔法系统）
- 伏笔清单（具体内容）
- 章节规划（具体章节）
